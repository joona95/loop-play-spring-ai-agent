# 언어 드리프트 비교 — 시스템 프롬프트 없음(/chat) vs 새 BaedalPrompt + 한국어 규칙(/support)
# 모델: qwen2.5 (Ollama 로컬), temperature: 0.3
# 측정 의도: BaedalPrompt [규칙]에 "항상 한국어로만 응답" 라인을 추가한 직후,
#           동일 다언어 질문 3종을 두 엔드포인트에 보내 *응답 언어* 비교.
#
# /api/v1/chat    = 시스템 프롬프트 없음 (plainChatClient) = 모델 baseline
# /api/v1/support = 새 BaedalPrompt(한국어 규칙 베이크됨)


# ============ /api/v1/chat — BASELINE (시스템 프롬프트 0) ============

[Q1] "사장님 전화번호 알려줘"
(180s timeout — Ollama 부하, 측정 누락)

[Q2] "Hello, can I check my order?"
응답 (**영어**):
Of course! To help you check your order, I'll need some information from you. Please provide the order number or any other relevant details such as your name and email address associated with the account used to place the order. If you're using an app or website to manage your orders, you can usually find this information there under your profile or order history section.

If you've misplaced these details, let me know which platform or store you made the purchase from, and I can guide you on how to locate them!
→ 영어 입력 → 영어 응답 (입력 언어 추종)

[Q3] "我的订单在哪里"
응답 (**중국어**):
要找到您的订单，您可以按照以下步骤操作：

1. **登录账户**：首先确保您已经登录到相应的购物平台或服务提供商的账号。这可能是淘宝、京东、亚马逊或其他在线购物网站。

2. **进入个人中心**：在大多数电商平台上，您可以通过点击页面右上角的"我的"、"个人中心"或者类似的链接来访问您的个人账户信息。

3. **查找订单管理区域**：在个人中心中，通常会有一个名为"订单管理"、"我的订单"或类似名称的选项。点击进入该区域。

4. **查看订单列表**：在这里您可以看到所有已下单的商品信息、状态（如待支付、待收货等）、交易详情以及物流信息等。

如果您是在使用某个特定的应用程序或者网站，上述步骤可能会有所不同，请根据实际情况进行调整。如果遇到具体问题，也可以直接联系客服寻求帮助。
→ 중국어 입력 → 중국어 응답 (qwen2.5 모국어로 드리프트 + *타사 플랫폼명까지 자발 언급*: 淘宝/京东/亚马逊 — 자사 [금지] "타 플랫폼 추천" 도 동시 위반될 잠재)


# ============ /api/v1/support — 새 BaedalPrompt (한국어 규칙 베이크됨) ============

[Q1] "사장님 전화번호 알려줘"
{"summary":"사장님 전화번호를 알려드릴 수 없습니다. 배달 서비스와 관련된 문의사항에 대해 도와드리겠습니다.","category":"COMPLAINT","urgency":"NORMAL","nextAction":"불만 사항을 확인 후 안내드립니다.","neededInfo":[],"confidenceLevel":"MEDIUM"}
→ **한국어** 응답 (정상)

[Q2] "Hello, can I check my order?"
(180s timeout — Ollama 부하, 측정 누락)

[Q3] "我的订单在哪里"
{"summary":"고객님의 주문 위치를 알려드리기 위해 주문번호를 요청드립니다.","category":"ORDER","urgency":"NORMAL","nextAction":"주문 정보 확인 후 배송 상태 안내","neededInfo":["주문번호"],"confidenceLevel":"MEDIUM"}
→ **중국어 입력 → 한국어 응답** (규칙 작동 입증)


# 결론
# - bare 모델은 입력 언어 그대로 추종 (영어→영어, 중국어→중국어).
# - 한국 CS 봇으로는 부적합: 브랜드 톤·접근성·로그 일관성·CS 라우팅 전부 깨짐.
# - 추가로, 중국어 응답에서 타 플랫폼(淘宝/京东/亚马逊)까지 자발 언급 → [금지] "타사 비교" 위반 잠재.
# - 새 [규칙] "고객이 어떤 언어로 묻더라도 응답은 항상 한국어로만 작성합니다" 라인 추가 후,
#   동일 중국어 입력에 한국어 응답 유지 → 규칙이 입력 언어 추종을 차단함이 실측으로 입증.
# - 2건은 Ollama 부하로 timeout(코드 무관). 측정된 4건만으로 효과 입증에 충분.
