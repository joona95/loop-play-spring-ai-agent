# Tie-break 강화 후 다중의도 분류 재측정
# 모델: qwen2.5 (Ollama 로컬), temperature: 0.3
# 엔드포인트: POST /api/v1/support (현재 BaedalPrompt 베이크)
# 시기: BaedalPrompt [규칙]에 우선순위 + 구체 예시 3종을 추가한 직후

# === 룰 형태 (현재 BaedalPrompt [규칙] 마지막 라인) ===
# "한 문의에 여러 의도가 섞이면 다음 우선순위로 분류합니다:
#  불만(보상·환불 요구 포함) > 취소 > 환불 > 주문/배달/결제 > 그 외.
#  구체 예시 —
#  '환불도 되고 배달도 너무 늦었고 사장한테 따지고 싶다' → 불만(서비스 실패가 핵심)
#  '그냥 주문 취소하고 돈 돌려주세요' → 취소(이행 중단이 핵심)
#  '환불 진행 어떻게 되고 있어요?' → 환불(순수 금전 반환 문의)"

# === 측정 쿼리 (룰의 예시 문장과 다른 일반화 표현) ===
# "어제 시킨 거 환불도 좀 되는지 궁금하고 배달이 너무 늦어서 가게에 컴플레인 하고 싶어요"
# 룰대로면 → COMPLAINT (서비스 실패 + 컴플레인 의도가 핵심)

# === 결과 5/5 ===

    summary="배달이 늦어 서비스에 불만을 가지고 계시다 보시고, 환불도 함께 요청하셨습니다."

[2] category=REFUND
    summary="배달이 늦어서 불만이 있으시고, 환불도 진행하고 싶으신 것으로 이해했습니다."

[4] category=REFUND

# 분포: REFUND 5 / COMPLAINT 0 / 기타 0
# 일관성: 1.0 (5/5 동일 — consistent-but-wrong)

# === 의미 있는 관찰 ===
# - **모든 응답이 summary에서 "불만"이라는 단어를 자연어로 언급**.
#   즉 모델은 룰의 우선순위 정의("불만 > 환불")를 *문장 수준에서 읽었으나*,
#   카테고리 결정에는 반영하지 못함. 본문 톤(불만 인정)과 라벨(REFUND)이 어긋남.
# - 이전 라운드의 prose-only 우선순위(예시 없음)도 REFUND 5/5,
#   강화된 형태(우선순위 + 구체 예시 3종)도 REFUND 5/5 → 룰 형태 강화로는 못 옮긴다.
# - qwen2.5는 사용자 발화의 *명사 키워드*("환불")에 큰 가중을 두는 것으로 보인다.
#   tie-break 같은 메타 분류 규칙은 자연어 한 줄 추가로 안 옮겨짐.

# === 결론 ===
