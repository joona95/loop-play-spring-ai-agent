판정
----------------------------------------------------------------
- order 수정 '전'(SimpleLoggerAdvisor 기본 order 0 < memoryAdvisor 10)이라면
  턴2 request도 [SystemMessage, UserMessage("그거 언제 도착?")] 둘뿐으로 찍혀 메모리가 "안 보였다".
- order 20(주입 후 로깅)으로 올리니 턴1 USER+ASSISTANT가 프롬프트에 들어간 게 그대로 관측된다.
- 즉 메모리는 처음부터 주입되고 있었고(응답의 맥락 해소·/messages 저장이 증거였다), '안 보인' 건
  관측 지점(로깅 advisor)의 위치 문제였음을 로그 구조 변화(User 1→2, Assistant 0→1)로 확정.
- 부수 관찰: 턴2는 getDeliveryStatus를 재호출하지 않고 메모리를 재서술(Finding 3 재현).
  턴1 응답엔 "라이der" 언어 드리프트가 그대로 메모리에 저장돼 턴2 프롬프트로 재주입됨(Finding 6).
